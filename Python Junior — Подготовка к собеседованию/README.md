# 🎯 Python Junior — Подготовка к собеседованию

Полноценный тренажёр для подготовки к техническому собеседованию на позицию **Junior/Junior+ Python-разработчика**.  
Вопросы, задачи-ловушки и практические упражнения — всё, что спрашивают на реальных интервью.

> Этот репозиторий содержит **10 тем**, охватывающих все ключевые области, которые проверяют на собеседованиях.  
> Каждая тема включает **теоретические вопросы с ответами**, **задачи-ловушки (что выведет код?)** и **практические упражнения**.

## 📚 Подробное описание тем

### 01 — Основы и устройство Python
Типы данных, изменяемость vs неизменяемость, передача аргументов (по ссылке или по значению?), интернирование строк и чисел, `is` vs `==`, shallow copy vs deep copy, PEP8.  
**Почему спрашивают:** Проверяют, понимаете ли вы, как Python работает «под капотом», а не просто пишете код.

### 02 — Функции и декораторы
Области видимости (LEGB), замыкания, `*args` / `**kwargs`, мутабельные аргументы по умолчанию (классическая ловушка!), декораторы (в т.ч. с параметрами), `functools.wraps`, генераторы, итераторы, `yield`.  
**Почему спрашивают:** Декораторы и генераторы — маркер уровня разработчика. Junior обязан понимать хотя бы основы.

### 03 — ООП под капотом
`__new__` vs `__init__`, MRO (Method Resolution Order), `super()`, магические методы (`__str__`, `__repr__`, `__eq__`, `__hash__`), абстрактные классы, миксины, дескрипторы, `@property`, `__slots__`.  
**Почему спрашивают:** ООП — основа большинства production-кодовых баз. Нужно знать не только синтаксис, но и механику.

### 04 — Структуры данных: глубокое погружение
Как работают словари (хеш-таблицы), почему ключи должны быть хешируемыми, сложность операций (Big O) для `list`, `dict`, `set`, `deque`. `collections` модуль: `defaultdict`, `Counter`, `OrderedDict`, `namedtuple`.  
**Почему спрашивают:** Выбор правильной структуры данных — ежедневная задача разработчика.

### 05 — Память, GIL и многопоточность
Управление памятью в Python (reference counting, garbage collector), что такое GIL и зачем он нужен, `threading` vs `multiprocessing`, когда что использовать, `concurrent.futures`.  
**Почему спрашивают:** Классический вопрос-фильтр. Кто не знает про GIL — скорее всего, не работал с нагруженными системами.

### 06 — Базы данных и SQL
`JOIN` (INNER, LEFT, RIGHT, FULL), `GROUP BY` + `HAVING`, индексы (зачем, когда замедляют), транзакции и ACID, нормальные формы, N+1 проблема в ORM, `SELECT` vs `SELECT DISTINCT`.  
**Почему спрашивают:** Почти любой бэкенд работает с БД. SQL — must have.

### 07 — Веб и сети (HTTP, REST API)
HTTP-методы (GET, POST, PUT, PATCH, DELETE), коды статусов (200, 301, 400, 401, 403, 404, 500), REST принципы, разница между cookies и сессиями, авторизация vs аутентификация, CORS, HTTPS.  
**Почему спрашивают:** Веб-разработка — основное направление для Python Junior.

### 08 — Асинхронность (asyncio)
Что такое Event Loop, корутины, `async`/`await`, `asyncio.gather`, `asyncio.create_task`, разница между sync и async, когда asyncio НЕ поможет, `aiohttp`.  
**Почему спрашивают:** Асинхронность всё чаще используется в production. Даже Junior должен понимать базовые концепции.

### 09 — Тестирование и Git
`unittest` vs `pytest`, фикстуры, моки (`unittest.mock`), parametrize, что тестировать, а что нет. Git: `merge` vs `rebase`, `cherry-pick`, решение конфликтов, `.gitignore`, `git stash`.  
**Почему спрашивают:** Код без тестов — не production-ready. Git — ежедневный инструмент.

### 10 — Живое кодирование (Live Coding)
Типичные задачи, которые просят решить прямо на собеседовании: разворот строки, FizzBuzz, поиск дубликатов, валидация скобок, палиндромы, Фибоначчи, анаграммы, слияние словарей.  
**Почему спрашивают:** Проверяют умение писать код в стрессовой ситуации и объяснять свои решения.

## 🚀 Как пользоваться

### Рабочий процесс

1. **Откройте `questions.md`** — попробуйте ответить на вопрос ДО того, как прочитаете ответ
2. **Закройте ответ** — мысленно сформулируйте свой ответ, потом сравните
3. **Разберите `tasks.md`** — попробуйте предсказать вывод кода БЕЗ запуска
4. **Решите `exercises.py`** — напишите код, запустите и проверьте
5. **Повторите через 2-3 дня** — интервальное повторение закрепляет знания

### Как имитировать собеседование

```
1. Засеките 45 минут
2. Откройте случайную тему
3. Ответьте на 5 теоретических вопросов вслух (!)
4. Решите 2 задачи из tasks.md без запуска кода
5. Напишите 1-2 решения из exercises.py
6. Оцените себя честно
```

## 💡 Советы для собеседования

### До собеседования
- 📝 Отвечайте на вопросы **вслух** — на интервью нужно уметь объяснять
- 🔁 Повторяйте слабые темы каждые 2-3 дня
- 💻 Решайте задачи на время (15 минут на задачу)
- 📖 Читайте чужой код — это тренирует навык code review

### На собеседовании
- 🗣️ Думайте вслух — интервьюер хочет видеть ход мысли
- ❓ Задавайте уточняющие вопросы — это признак опыта
- 🧪 Проверяйте edge cases — пустой ввод, None, отрицательные числа
- 🤷 Если не знаете — честно скажите и предположите

### Типичные ошибки кандидатов

| Ошибка | Как избежать |
|--------|-------------|
| Зубрёжка без понимания | Объясняйте концепции своими словами |
| Молчание при решении | Комментируйте каждый шаг вслух |
| Паника при незнакомом вопросе | Разбейте на части, начните с того, что знаете |
| Игнорирование edge cases | Всегда спрашивайте: «А что если ввод пустой?» |
